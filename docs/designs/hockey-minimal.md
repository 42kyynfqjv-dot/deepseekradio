# Hockey League Engine — MINIMAL-STATE-INCREMENTAL Design

Lens: smallest schema, fewest moving parts, every component shippable and testable
tonight, migration and fallback trivially safe, every byte of state paid for by a
listener-audible gain.

## 1. Executive summary

- **season.json stays the spine, unchanged in shape.** Standings, games, out-list,
  slates keep their exact v1 schema. Everything new lives in small **sidecar files**
  under `data/league/`; if a sidecar is missing, every code path degrades to v1
  behavior. Fallback is "flip one flag" — v1 never stops being valid.
- **Derive, don't store.** Players, schedule, and off-air box scores are pure
  deterministic functions of `(season, seed)`. We persist them once (cheap insurance
  against code drift), but state loss is always recoverable from seeds. Box scores
  are pruned to 21 days; accumulated player stats live in one compact aggregate file.
- **One model, three speeds.** The existing calibrated piecewise-Poisson engine gains
  nothing new except *weighted scorer draws* and a *second assist*. Off-air games run
  the same `_sim_span` in event mode (names on), giving every game a real box score
  for ~3ms.
- **League calendar = wall clock, identity mapping.** 1 wall day = 1 league day,
  82 games in ~26 weeks. No translation layer, no drift, date-seeding untouched.
- **Rollover moves out of `record_live` into `tick`, air-gated** — the known bug dies
  as a side effect of the phase machine.
- Total new steady-state disk: **< 1 MB/season.** Total new `tick()` cost: **< 100 ms
  on a day boundary, ~1 ms otherwise.**
- Ship order tonight: schedule → players → boxscore → stats → briefs → gate+migrate.
  Economy and playoffs are designed now, gated separately, and not needed Saturday
  (deadline is months of league time away).

## 2. State schema

All files atomic tmp+replace with `.bak` copy (existing `_save` pattern). Sharded
**per season** (`-s1`, `-s2`, …); previous season's files kept whole, older seasons
collapsed into `records.json`.

### season.json (existing — extended, never reshaped)

```json
{
  "season": 1, "game_no": 2, "sim_through": "2026-07-10",
  "league": {"mtl": {"w":1,"l":1,"otl":0,"streak":1,"gp":2,"rw":1,"row":1}, "...": {}},
  "recent_opponents": [], "last_result": "...",
  "games": {"2026-07-08": {"...": "unchanged v1 game dicts"}},
  "slates": {"2026-07-10": [["tbr","hfx",4,2,false]]},
  "out": {},
  "v2": {"phase": "regular", "start": "2026-07-05", "rolled_pending": false}
}
```

New keys (`rw`, `row`, `v2`) are additive; v1 code ignores them. `out` retained for
fallback; v2 keeps it mirrored (names only) while the authoritative injury table is
`players` sidecar `out2`.

### data/league/players-s1.json (~260 KB)

```json
{
  "schema": 1, "season": 1,
  "players": {
    "mtl-01": {"name": "Doug Bouchard", "team": "mtl", "pos": "C", "slot": "F1",
               "by": -26, "ov": 0.72, "sh": 0.60, "pl": 0.76, "dur": 0.55,
               "aav": 8.4, "yrs": 3, "aired": true},
    "mtl-22": {"name": "Curtis Vachon", "team": "mtl", "pos": "G", "slot": "G1",
               "by": -28, "ov": 0.68, "sh": 0.0, "pl": 0.0, "dur": 0.7,
               "aav": 6.1, "yrs": 2, "aired": true}
  },
  "reserve": {"mtl": ["mtl-r1", "mtl-r2", "mtl-r3", "mtl-r4"]},
  "out2": {"mtl-07": {"until": "2026-07-19", "note": "week-to-week, lower body",
                       "games": 4, "ir": true}},
  "callups": {"mtl": ["mtl-r1"]},
  "retired": []
}
```

- 32 teams × (23 active + 4 reserve) ≈ **864 players** × ~180 B ≈ 160 KB, plus
  out/callups. Age is derived: `age = 27 + (season - 1) - by`… concretely
  `by` = birth-season offset; `age(p, season) = season - p["by"]` (season 1 player
  born `by=-26` is 27). No age field to keep in sync.
- `slot` ∈ {F1..F4 lines ×3, D1..D3 pairs ×2, G1, G2} fixes lines/pairings without a
  separate structure.
- `ov/sh/pl` in [0,1]: overall, shooting bias, playmaking bias — the **entire**
  attribute model. Depth-curve fidelity comes from slot-tiered `ov` at mint plus the
  weighted-draw exponent (§11), not from more attributes.

### data/league/schedule-s1.json (~55 KB, immutable once written)

```json
{"schema": 1, "season": 1, "start": "2026-07-05",
 "days": {"2026-07-11": [["mtl","nyg","AIR"], ["tbr","hfx"], ["ssk","wpg"]]},
 "playoff_start": "2027-01-09"}
```

1,312 rows × ~30 B. Generated by a pure function; stored so a scheduler code change
mid-season can never reroute already-published context. `"AIR"` tags the broadcast
game (Wed/Sat, exactly one, always a tracked team).

### data/league/stats-s1.json (~150 KB)

```json
{"schema": 1, "season": 1,
 "skaters": {"mtl-01": [12, 5, 9, 4, 1, 0]},
 "goalies": {"mtl-22": [8, 5, 2, 1, 244, 226, 1]}}
```

Skater array: `[gp, g, a, pim, gwg, hat]`. Goalie: `[gp, w, l, otl, sa, sv, so]`.
Points, SV%, GAA derived. One flat file, folded incrementally — never recomputed.

### data/league/box/2026-07-12.json (~8 KB/night, pruned to 21 days)

```json
{"date": "2026-07-12", "games": [
  {"home": "tbr", "away": "hfx", "final": [4,2], "ot": false, "so": false,
   "goals": [{"t":"h","period":2,"clock":"5:12","scorer":"tbr-03",
              "a1":"tbr-01","a2":null,"str":"PP"}],
   "shots": [31,27], "goalies": {"h":"tbr-22","a":"hfx-22"},
   "stars": ["tbr-03","tbr-22","hfx-04"],
   "injuries": [{"pid":"hfx-09","note":"day-to-day, upper body"}]}]}
```

This is the around-the-league narration feed. 21 days × 8 KB ≈ 170 KB ceiling.
Older boxes are re-derivable from seeds if ever needed.

### data/league/coaches-s1.json (~6 KB)

```json
{"coaches": {"mtl": {"name":"Marcel Demers","style":"defensive","mod":0.015,
                      "hired_day": 0}},
 "trainers": {"mtl": {"name":"Ruth Halloran","heal":0.92}}}
```

### data/league/transactions-s1.json (~25 KB, append-only, per-season)

```json
{"tx": [{"day": 87, "date": "2026-09-30", "type": "trade", "from": "tbr",
         "to": "far", "out": ["tbr-11"], "in": ["far-14"], "note": "cap dump"},
        {"day": 60, "type": "coach_fired", "team": "bis", "old": "...", "new": "..."}]}
```

### data/league/records.json (append-only, a few KB, never pruned)

```json
{"seasons": {"1": {"cup": null, "presidents": null,
                    "art_ross": {"pid":"...","name":"...","pts": 0},

 "trophy": "TBD (§13)"}
```

**Pruning story:** box shards 21 days; slates 30 days (unchanged); games 90
(unchanged); per-season sidecars kept for current + previous season only, older
seasons survive as one `records.json` entry. Steady-state footprint: **~700 KB.**

## 4. Interface preservation

- **`tonight_live(date)`** — same signature, same idempotence, same game-dict keys.
  `rosters` keeps the exact `{home/away: {skaters: [names], goalie: name}}` shape;
  the skater list grows from 8 to 18 (L1 first, so `_draw_skater`'s "first 3 are
  stars" instinct and the booth's habit of leading with the top line both survive).
  Additive keys: `rosters[side]["weights"]`, `["ids"]`, `["backup"]`, `game["lines"]`.
  `strength_home/away` now derived from the roster but still floats in [0.30, 0.70];
  `refs`, `subplot`, `arena`, `attendance`, `game_no`, `rivalry`, `season` untouched.
- **`record_live(date)`** — same signature/idempotence. Additionally folds the live
  log's goals into `stats-s{n}.json` (player names → ids via tonight's `ids` map) and
  samples v2 injury durations. **No longer rolls the season** (§8).
- **`tick(date)`** — same signature; §6 algorithm; sub-second steady state preserved
  by a fast path.
- **`export(path)`** — same air-gating logic verbatim; output JSON gains additive
  keys (`leaders`, `around[].scorers`, `playoffs`) — the site ignores unknown keys.
- **`pregame_brief` / `postgame_brief` / `context_pairs`** — same signatures, richer
  content (§10). `context_pairs` additionally returns today's and yesterday's box
  finals so the intermission scores desk can quote them without scoreguard
  replacement.
- **Guard fact shapes** — `build_facts` reads `game["rosters"]` lists and `refs`
  exactly as today; longer skater lists just widen `names_ok`/`surnames`. `assist2`
  names are roster names, already in `names_ok`. `_ev_text` gains one optional
  clause for the second assist. Nameguard's `pool_ok` unchanged (players still mint
  from `FIRST_NAMES`/`LAST_NAMES`).

## 5. League calendar mapping

**1 wall-clock day = 1 league day. Identity. No translation layer.**

Justification: the station broadcasts exactly two tracked games a week (Wed/Sat) —
that is a hard floor on season length. A tracked team airing 2/week plus ~1.2 off-air
games/week plays **3.15 games/week**, which is *precisely NHL cadence* (grounding:
3.0–3.2/week). So the maximally compressed calendar that keeps every broadcast a real
schedule game **is** the NHL calendar, 1:1: **82 games in ~26 weeks**. Any further
compression makes broadcast games a rounding error of the season (fidelity loss:
standings would move implausibly between shows), and any league-day≠wall-day mapping
adds a translation table that date-seeding, `sim_through`, the scorebug, and the
listener's own sense of "last Saturday" would all have to agree on. Identity costs
zero state and zero bugs.

Concrete season-1 numbers (start 2026-07-05, migrated mid-flight):
- Regular season: remaining ~178 days → finale ~**2027-01-06**.
- League slate: 32 teams × 3.15/wk ÷ 2 ≈ **7–8 games/night** (vs ~10 today — slightly
  quieter, correct per grounding). Christmas break Dec 24–26 baked into `assign_days`.
- Playoffs: **2027-01-09 → ~mid-Feb** (best-of-7, games every 2 days, tracked series
  pinned to air nights ±1 day). Offseason 14 days (draft, FA, camp as news beats).
  Season 2 opens ~**2027-03-01**. Cycle ≈ 8 months — two Cup runs every ~16 months.
- **Tracked-team matrix exception:** mtl and nyg are in different divisions, so the
  NHL matrix gives them 3 meetings — but `_RIVALRY_EVERY = 7` needs ~7. The league
  canonizes a "**Crossover Series**": mtl–nyg meet **8 times** (4H/4A), paid for by
  dropping one game from five of each team's other buckets. The other 30 teams get
  the exact NHL matrix. In-universe flavor, zero extra state, rivalry cadence
  preserved exactly.
- Back-to-backs emerge naturally from `assign_days` (target 10–16 sets/team); the
  `b2b` flag feeds a −4% strength dip (§11).

## 6. The nightly tick algorithm

`tick(date)`, every main-loop pass:

1. **Fast path** (every pass, ~1 ms): load `season.json` (already done today); if
   `sim_through == date` and no dirty flag → run `_reconcile`, throttled `export()`
   (skip if <30 s since last), return. Sidecars are *not even opened*.
2. **Day boundary** (once per day, or catch-up loop): for each missing day `d` in
   `(sim_through, date]`:
   a. Load sidecars once for the whole loop (players 260 KB + stats 150 KB + schedule
      55 KB ≈ 5 ms parse total).
   b. **Returns:** scan `out2`, clear healed players, send call-ups back down.
   c. **Slate:** `schedule["days"][d]`; skip the `"AIR"` game (the live engine owns
      it); for playoffs phase use `schedule_series()` instead.
   d. Per game: `dress()` both teams (out-filtered, call-ups applied),
      `team_strength()` (coach mod, B2B dip), then
      `sim_box(..., random.Random(f"slate2:{season}:{d}:{hk}-{ak}"))` — **per-game
      seeds**, so a slate is order-independent and self-healing game by game.
   e. Fold: `_apply()` standings (+`rw`/`row`), `fold_box()` stats, box injuries →
      `out2` via `sample_injury()` (log-normal, median 7 d), mirror names into v1
      `out`. Append v1-shaped row to `st["slates"][d]` (fallback stays warm).
   f. `economy.run_day()` — seeded `f"econ:{season}:{d}"`: trade check (thinned
      Poisson, deadline-week spike), coach-firing check (λ≈5.5/season, Nov–Feb
      analog window), emits transactions + news lines.
   g. `milestones()` → `data/league/news-lines.json` for other shows' scores desks.
   h. Write `box/{d}.json`; prune box>21 d, slates>30 d.
3. **Phase transitions, air-gated** (§8).
4. Save dirty files (atomic + .bak), `_reconcile`, `export()`.

**Cost on 2 vCPU:** `sim_box` is `_sim_span` with events — a few hundred exponential
draws — measured-class **~2–4 ms/game** in CPython; 8 games + folds + JSON writes ≈
**60–100 ms per day boundary**. Catch-up after downtime: ~100 ms/day → 30 days ≈ 3 s;
bounded by chunking at **45 days per tick pass** (the loop yields and resumes next
pass; standings converge in ≤3 passes even after a multi-month outage). Memory:
< 5 MB transient. Kokoro is undisturbed.

## 7. Migration plan (live 4-day-old season 1)

One script, **`scripts/migrate_league_v2.py`**, idempotent, runs with the gate OFF:

1. Read `season.json`. **Never writes it** except to add the `v2` key at cutover.
2. **Players:** for each team, call `season._roster(key, 1)` — the same seed that
   produced the aired names — and pass those 9 names to `mint_league()` as the
   protected core: the 8 skaters become F1 line + F2 wing + D1 pair by mint order,
   the goalie becomes G1. Doug Bouchard and Rene Tremblay keep their names, teams,
   and star billing forever. 14 new actives + G2 + 4 reserves mint around them.
   `ov` tiers are solved so `team_strength()` ≈ v1 `_strength(key, 1)` within ±0.01 —
   **the competitive landscape aired to date is preserved, not re-rolled.**
3. **Standings:** copied by reference — the file isn't touched. `rw`/`row` initialize
   from `w` (season-1 approximation, noted; exact from season 2).
4. **Schedule:** `assign_days(build_matchups(1), 1, "2026-07-05",
   gp_played={from standings}, played_pairs={from st["slates"]+st["games"]})` —
   builds only the **remaining** ~80 games/team so totals hit exactly 82 and nobody
   replays anyone implausibly. (Past 4 days' opponents weren't matrix-constrained;
   season 1's matrix is approximate — inaudible — and exact from season 2.)
5. **Stats back-fill:** aired games 1–2 → fold real scorers from the surviving
   `livegame-*.jsonl` logs (names→ids). Past off-air slates (stored finals) →
   `box_from_final()` with seed `f"retro:{season}:{day}"` allocates goals to the
   minted rosters consistent with each published final. Stats now agree with every
   number ever aired, and `gp` sums match standings.
6. **Coaches/economy:** mint from name bank; contracts solved to the grounding cap
   shape (top 6–8 deals ≈ 60% of $95.5 M) with team payroll between floor and
   ceiling.
7. **Verify** (hard gate, §8): every aired name present at its team; standings dict
   deep-equal pre/post; Σ player gp == Σ standings gp × roster factor; schedule totals
   = 82/41H/41A per team; rivalry games = 8 on every-7th-broadcast slots.

The brief is right that migration is nearly free — because the aired names were
seed-derived, the extension re-derives them from the same seed rather than parsing
transcripts.

## 8. Feature gate + cutover + instant fallback

- **Gate:** `data/league/ENABLED` flag file (checked once per `tick`; also
  `config.yaml: league_v2: true` for belt-and-braces). `season._v2_on()` requires
  flag AND all sidecars parse. Any sidecar corruption → loud print, v1 path.
- **Cutover (Saturday, before 20:00):**
  1. Fri night: land modules + tests (gate off; live air untouched — v1 code paths
     byte-identical except the rollover fix and the additive `_draw_skater` arg).
  2. Sat morning: run migration, run `scripts/verify_league.py` (migration checks §7
     + calibration smoke §11 + a full offline dry-run of `tick` over a copied state
     directory + golden-output check that `pregame_brief` renders and scoreguard
     accepts a synthetic beat against the new facts).
  3. Sat ~14:00: `touch data/league/ENABLED`. Watch one manual `tick` + `export`;
     eyeball league.json and the pregame sheet.
  4. 20:00 broadcast runs v2. The live engine sees a bigger roster + weights —
     its only behavioral delta.
- **Instant fallback:** `rm data/league/ENABLED`. Because v2 mirrors every fold into
  the v1 fields of `season.json` (standings, slates rows, `out` names), v1 resumes
  mid-season with zero repair. Sidecars stay on disk, ignored, ready to re-enable.
- **Rollover air-gating (the bug):** rollover leaves `record_live` entirely. The
  phase machine in `tick`: when the finale is complete → `phase="playoffs"`; when
  `champion()` → `phase="offseason"`, `rolled_pending=true`; the actual reset to
  season N+1 executes only when `rolled_pending` AND today's live log (if any) has
  `final_air_at` in the past AND no unrecorded game exists. A season can therefore
  never zero itself under a broadcast that hasn't reached listeners. This fix ships
  in **both** modes Friday night. Test: fold a final with `final_air_at` in the
  future → assert `tick` refuses to roll.

## 10. Broadcast-layer contract

All sheets are strings/dicts built by `briefs.py` from sidecars — code-owned facts,
LLM narrates, guards verify (names are roster names ⊂ `names_ok`; scores enter
`context_pairs`).

- **Pregame** — `pregame_brief(game)` (same function, richer): existing content +
  `LINES:` (four F lines, three D pairs, starting/backup goalies), `INJURY REPORT:`
  (named, with grounded durations: "Petrenko, week-to-week, out ~3 more games"),
  `LEADERS:` (top-5 points, top goalie SV%), `MILESTONE WATCH:`, `AROUND TONIGHT:`
  (matchups only — those games resolve at day-sim, never pre-narrated as finals
  before their own night is done).
- **Intermission / around-the-league** — `intermission_sheet(date) -> dict`:
  `{"around": [{"home","away","final":[..],"ot","scorers":["Toivo Ostberg (2)",...],
  "winning_goalie":"...","shots":[..]}], "leaders":[...], "race_note": "..."}` —
  **named scorers from `box/{date-1}.json` and any completed earlier games**; wired
  into the existing intermission-2 "look around the league" beat. Every pair goes
  into `allow_pairs`.
- **Postgame interviews** — `postgame_brief(game, final)` (same signature) appends a
  `QUOTE GROUNDING` block: goal list with strengths/clocks, save counts per goalie,
  each star's season line ("Bouchard now has 21 on the year"), coach's record — so
  coach/player quotes are verifiable against the box; scoreguard's tallies and
  `_STAT_CTX` already tolerate stat pairs.
- **Scores desk (other shows)** — `scores_desk(date, n=5) -> str`: "Last night in the
  league: Regrets 4, Fog Advisories 2 — Ostberg twice; …" written to
  `data/league/news-lines.json`; `news.py`/morning shows read it like any wire copy.
- **Website league page** — `export()` gains `"leaders"` (points/goals/SV% top-5 with
  team), `"around"` rows gain `"scorers"`, `"playoffs"` (bracket when phase), and
  per-team `"pts%"`. Air-gating logic untouched.
- **OT-past-window signal** — `data/ot-signal.json`:
  `{"date":"...","live": true, "period":"OT", "score":[2,2],
    "expected_min_more": 6, "written_at": 1789...}` — written when the live game is
  unfinal within 10 min of window end, cleared on final. Scheduler/site poll it;
  the site shows "still going past the hour", the next show's open can acknowledge it.

## 11. Statistical calibration plan

**Constants and where they're tuned:**
- Team level is already calibrated (BASE_EV, OT_MULT, …) — untouched.
- New knobs: scorer-draw weight `w_i = (ov_i · (0.5 + sh_i))^γ` with γ tuned so slot
  PPG hits the depth curve (L1 0.80–1.15 → L4 0.10–0.25); assist chain
  `P(a1)=0.90, P(a2|a1)=0.65` → A:G = 0.90+0.585 ≈ **1.49** (target 1.45–1.55),
  playmaker-weighted picks push top A:G to 2.0+; goalie start split: G1 starts with
  p=0.73 (→ 60/22 GP); B2B strength dip ×0.96; coach `mod` ∈ [−0.02, +0.02];
  injury log-normal (μ=ln 7, σ=1.25) checked against MGL 195±85.
- **Verifier:** `scripts/calibrate_league.py --seasons 50` sims 50 full seasons
  (65k games × ~3 ms ≈ 3.5 min, runnable on the box off-peak or dev laptop) and
  asserts every consolidated target: GF/game 6.1±0.3; OT 19–23%; SO 9–12% of games;
  home win 52–56%; shutouts 7–8% of games; top team 112–120 pts, cutline 92–97,
  floor 50–58; Art Ross 118–148; Richard 48–70; 100-pt scorers 5–10; 40G 8–14;
  20G 110–135; league SV% .899–.906; Vezina-tier .920–.930; hat tricks 75–120;
  max win streak 6–10 typical; MGL band; A:G band. Output is a pass/fail table;
  **cutover step 2 requires the smoke subset (5 seasons, wide bands) to pass, and
  Sunday's full run gates the economy flag.**
- Drift protection: the table is a pytest (`tests/test_calibration.py`, marked slow)
  so any future constant change re-proves the envelopes.

## 12. Coaches / trainers / economics model

Deliberately shallow-but-audible — one scalar of effect per role, everything else is
narration:
- **Coach:** `mod` (±0.02 team strength ≈ ±3 wins/season) + `style` (flavor word the
  booth uses). Firing: daily seeded check, Poisson λ≈5.5/season concentrated in the
  Nov–Feb analog; trigger biased toward teams >6 pts under expectation (expectation =
  preseason payroll rank). New hire minted from the name bank; transaction + news
  line emitted ("the Broken Thermostats have relieved Gord Sopel of his duties").
- **Trainer:** `heal` ∈ [0.85, 1.15] multiplies injury durations — audibly different
  franchises ("Halifax gets guys back early, it's the trainer").
- **Contracts/cap:** each player `{aav, yrs}`; ceiling $95.5 M, floor 0.74×,
  min $0.775 M, max hit 20% of ceiling; payroll shaped to grounding (top 6–8 deals
  55–65%). `yrs` decrements at rollover; expiring: age<27 → re-sign at formula
  (ov-scaled, ±seeded noise), else 15% chance walks to FA pool → seeded FA
  redistribution in offseason, cap-validated.
- **Trades:** `run_day` proposes cap-legal 1-for-1 (occasionally 2-for-1) swaps
  between a seller (>8 pts out, expiring vets) and a buyer (contender, cap room);
  thinned to 60–100/season with a deadline-day spike of 15–25 (deadline = first
  Friday of month 8 of the season). Tracked-team trades never touch an `aired:true`
  core player without the owner flipping `allow_tracked_trades` — aired names are
  canon and stay put by default.
- **Draft:** offseason, 2 rounds narrated (7 stored), lottery over the 16
  non-playoff teams with grounding odds (18.5%→0.5%, max 10-spot fall); picks become
  reserves with high-variance `ov`.

## 13. Playoffs + trophy

**Format (per grounding, exactly):** 16 qualifiers — top 3 of each of the 4 divisions
+ 2 wild cards per conference (our 2 conf × 2 div × 8 maps 1:1). Fixed bracket, no
reseeding: A1–WC2, B1–WC1, 2–3 within division. All rounds best-of-7, 2-2-1-1-1,
home ice by pts% (tiebreak: pts% → RW → ROW → W → H2H, which is why `rw`/`row` joined
the standings dict). Off-air series sim nightly via `sim_box` (games every 2 days);
a tracked team's series games are pinned to Wed/Sat ±1 and broadcast live by the
existing engine — playoff games are just games with `game["playoff"] = {"round":2,
"game":5, "series":[3,1]}` (additive key; briefs render "elimination game" stakes).
Bracket state lives in `data/league/playoffs-s{n}.json` (~4 KB). Cup winner and all
award winners append to `records.json`.

**Trophy shortlist** (owner rejected "Apology Cup"):
1. **The Halfway Cup** — named for the station's own town; every mention of the
   trophy advertises the universe's center of gravity.
2. **The Long Winter Cup** — deadpan northern fatalism; "who survives the Long
   Winter" writes its own playoff copy.
3. **The Thibodeau Trophy** — a mythical league founder from the existing surname
   bank; instantly retro-fits the league with decades of implied history.
4. **The Frequency Cup** — the station's name on the silverware; clean cross-promo
   every time the booth says it.
5. **The Boreal Lantern** — evocative, wintry, and unusually *not a cup*, which is
   exactly the kind of detail a fake league would never dare and a real weird league
   absolutely would.
